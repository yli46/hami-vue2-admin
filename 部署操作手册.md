# hami-vue2-admin 部署操作手册

> 目标：把本仓库部署到 `hami.vercel.app/freight/`（与老仓库共用主域名）。
> 已完成：本地脚手架 + git init + 第一次 commit。
> 你要做：创建 GitHub 仓库 → push → 接 Vercel → 改老仓库 vercel.json。

---

## 第一步：在 GitHub 创建新仓库（约 2 分钟）

打开浏览器：

```
https://github.com/new
```

填写：

| 字段 | 值 |
|---|---|
| Owner | `yli46`（与 hami-vue-admin 同账号） |
| Repository name | `hami-vue2-admin` |
| Description | 哈密煤运廊道一体管控平台 · 汽运详设新页面（Vue2 + Element-UI） |
| Visibility | Public（与 hami-vue-admin 一致；如需私仓选 Private） |
| Initialize this repository | **全部不勾**（不要 add README / gitignore / license — 本地已有） |

点击 **Create repository**。

页面会跳转到一个空仓库，给出 push 提示。**忽略页面提示**，按下面命令操作。

---

## 第二步：本地 push 到 GitHub（约 1 分钟）

```bash
cd /Users/Pcitc/Hami_Project/hami-vue2-admin
git remote add origin https://github.com/yli46/hami-vue2-admin.git
git push -u origin main
```

如提示要登录，输入你的 GitHub 用户名和 Personal Access Token。

push 完成后，刷新 GitHub 仓库页面应能看到所有文件。

---

## 第三步：Vercel 导入项目（约 3 分钟）

打开浏览器：

```
https://vercel.com/new
```

如未登录，先用 GitHub 账号登录。

在 **Import Git Repository** 页面：

1. 找到 `yli46/hami-vue2-admin` 仓库
2. 点击右侧 **Import**

进入项目配置页面：

| 配置项 | 值 |
|---|---|
| Project Name | `hami-vue2-admin`（默认即可） |
| Framework Preset | **Vite**（Vercel 会自动识别） |
| Root Directory | `./`（默认） |
| Build Command | `npm run build`（默认即可，已在 vercel.json 指定） |
| Output Directory | `dist`（默认即可，已在 vercel.json 指定） |
| Install Command | `npm install`（默认） |
| Environment Variables | 不填 |

点击 **Deploy**。

等 30-60 秒，构建完成后会显示部署 URL，类似：

```
https://hami-vue2-admin.vercel.app
```

或

```
https://hami-vue2-admin-xxxx.vercel.app
```

**重点**：Vercel 给的默认域名记下来，下一步要用。如果带后缀（-xxxx），可以在 Vercel 项目 Settings → Domains 里添加自定义域名 `hami-vue2-admin.vercel.app`（如果该名字已被占用，用默认即可）。

## 第四步：验证新仓库部署正确（约 1 分钟）

```
https://hami-vue2-admin.vercel.app/freight/
```

应该看到：

- 右侧主区域 8 张模块卡片

如果访问 `https://hami-vue2-admin.vercel.app/`（不带 /freight/）会 404，**这是正常的**——本仓库设计为只在 /freight/ 路径下提供服务。

## 第五步：修改老仓库 vercel.json，绑定子路径（约 2 分钟）

打开 `/Users/Pcitc/Hami_Project/hami-vue-admin/vercel.json`，**完整替换**为以下内容：

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "rewrites": [
    { "source": "/freight", "destination": "https://hami-vue2-admin.vercel.app/freight/" },
    { "source": "/freight/(.*)", "destination": "https://hami-vue2-admin.vercel.app/freight/$1" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

**关键点**：
- 第一条 / 第二条新加的 rewrite **必须放在第三条 SPA fallback 之前**（Vercel 按顺序匹配）
- 如果 Vercel 给你的 URL 不是 `hami-vue2-admin.vercel.app`，把上面两个 destination 里的 URL 替换成你的实际 URL

提交并推送：

```bash
cd /Users/Pcitc/Hami_Project/hami-vue-admin
git add vercel.json
git commit -m "chore: rewrite /freight/* to vue2 subapp"
git push
```

Vercel 自动部署，约 30-60 秒后老仓库的 hami.vercel.app 上 rewrite 生效。

## 故障排查

| 现象 | 可能原因 | 排查 |
|---|---|---|
| `/freight/` 404 | rewrite 没生效 | 检查老仓库 vercel.json 是否已部署 |
| 资源加载失败（CSS/JS 404） | base 路径不对 | 检查新仓库 vite.config.js base = '/freight/' |
| 路由刷新 404 | SPA fallback 缺失 | 检查新仓库 vercel.json rewrites 是否包含 /freight/(.*) → /freight/index.html |
| 跳转到 `hami-vue2-admin.vercel.app` 而非 `/freight/` | 链接写绝对域名 | 用 `<router-link to="/path">` 而非 `<a href>` |
| 登录态丢失 | 跨仓库 cookie 没共享 | localStorage 同域可读，确认两侧用 localStorage 而非 sessionStorage |
