# 前端基线对照清单

> **基线**：互联网事业部 v2 设计原则提炼 + prototype-standard.css。
> **用途**：每次新加页面 / 改样式前对照。每条违反必须改，不允许 case-by-case 自由发挥。
> **来源文件**：`哈密煤化工标准流程 2/哈密煤运廊道一体管控平台设计原则提炼.md`、`templates/prototype-standard.css`

---

## A. 整体风格底线（5 条）

- [ ] **A1**：不堆叠彩色卡片、不用重阴影浮层、不做营销式装饰块（v2 § 3.2）
- [ ] **A2**：信息层级靠版式（布局 / 间距 / 边框 / 留白），不靠装饰（v2 § 3.2）
- [ ] **A3**：一个页面只保留一个主要视觉焦点（v2 § 3.2）
- [ ] **A4**：状态通过轻量标签（el-tag）表达，禁止自定义彩色徽章 / 重色块铺陈（v2 § 3.2 / § 6.4）
- [ ] **A5**：后台页保留稳定边距，**不要把内容塞满整屏**（v2 § 3.3）

## B. 颜色体系（4 条）

- [ ] **B1**：主色 `#409EFF`（导航 / 主按钮 / 焦点态 / 激活态 / 链接态）（v2 § 3.1）
- [ ] **B2**：标题/品牌蓝 `#24558A`，选中浅蓝 `#E7F3FF`（v2 § 3.1）
- [ ] **B3**：白色负责卡片、表单、表格、弹窗、主工作区；灰色负责边框、背景层次、辅助文案；红色仅用于失败/删除/警告（v2 § 3.1）
- [ ] **B4**：颜色一律用 prototype-standard.css 的 CSS 变量（`var(--primary)` 等），不写硬编码色值

## C. 组件优先级（5 条）

- [ ] **C1**：基础组件**优先**用 Element UI（el-form / el-input / el-select / el-button / el-card / el-table / el-tree / el-dialog）（v2 § 4.1）
- [ ] **C2**：保留 Element UI 组件语义（按钮 primary / plain / text 三层权重）（v2 § 4.2）
- [ ] **C3**：行内操作 `el-button type="text"`，主操作 `type="primary"`，次操作 `plain`（v2 § 4.2 / § 5.4）
- [ ] **C4**：**不**重写组件结构、**不**绕开 Element UI DOM 自造一套（v2 § 4.1）
- [ ] **C5**：定制范围限于"轻改造"——圆角、边框、输入框高度、表头背景、卡片阴影、内边距、hover 态（v2 § 4.3）

## D. 页面模板（4 条）

- [ ] **D1**：列表页 = **查询卡 + 表格卡 + 分页**（顶搜索 / 中操作 / 主表 / 底分页）（v2 § 5.2）
- [ ] **D2**：左树右表 = 左树定位 / 右表结果与操作 / 中间稳定间距（v2 § 5.3）
- [ ] **D3**：编辑用弹窗或单卡表单，底部明确操作区（v2 § 7.4）
- [ ] **D4**：同类页面共用同类模板，**不频繁发明新结构**（v2 § 3.4）

## E. 交互规则（4 条）

- [ ] **E1**：删除前**必须**二次确认（this.$confirm）（v2 § 5.4）
- [ ] **E2**：危险操作通过对话框和文案强化（v2 § 5.4）
- [ ] **E3**：主操作前置 / 次操作收敛 / 危险操作确认 / 搜索-结果-详情分区清晰（v2 § 7.5）
- [ ] **E4**：保持 Element UI 原生交互语义和用户预期（v2 § 7.5）

## F. 卡片与表格细节（4 条）

- [ ] **F1**：卡片基于 el-card，**圆角约 2px，阴影基本去除**（v2 § 6.1）
- [ ] **F2**：禁止大圆角、重阴影、毛玻璃式卡片（v2 § 6.1）
- [ ] **F3**：表头底色偏浅灰蓝 `#F1F4F6`，行高紧凑（v2 § 6.4 / prototype-standard.css `--table-header-bg`）
- [ ] **F4**：宽表格允许横向展开，**不强行压成窄卡片**（v2 § 6.4）

## G. CSS 编码规范（3 条）

- [ ] **G1**：通用样式放全局 prototype-standard.css，避免每页各改一套（v2 § 4.3）
- [ ] **G2**：**禁用** `::v-deep` / `>>>` / `!important` 覆盖 Element UI 内部样式（v2 § 4.1 § 4.3）
- [ ] **G3**：组件 prop 优先（如 el-menu 的 `background-color` / `text-color` 等 prop），不要 deep selector 强改（v2 § 4.1）

## H. 反模式（不允许出现，10 条）

| # | 反模式 | 违反 v2 条款 |
|---|---|---|
| H1 | grid 堆叠 6+ 个统计卡 / 营销看板式拼贴 | § 3.2 / § 8 |
| H2 | hover 时卡片放大 / 上浮 / 阴影加重 | § 3.2 / § 6.1 |
| H3 | 自定义彩色徽章（绿橙红背景）替代 el-tag | § 3.2 / § 6.4 |
| H4 | 渐变背景 / 毛玻璃 / 大图标装饰 | § 3.2 / § 6.1 |
| H5 | 圆角 ≥8px / shadow ≥0 1px 8px | § 6.1 |
| H6 | `::v-deep` / `>>>` / `!important` 覆盖 Element UI | § 4.1 § 4.3 |
| H7 | 删除按钮无二次确认 | § 5.4 |
| H8 | 同类页面 A 用列表 B 用卡片 | § 3.4 |
| H9 | 数字大屏 / 进度条堆 / KPI 仪表盘式装饰 | § 8 |
| H10 | 响应式时改变骨架结构（如左树折叠成下拉菜单） | § 3.4 |

## I. 自审流程

每次新加 / 修改一个 .vue 文件后，对照下面 5 个问题：

1. 这个页面属于 v2 三种模板里的哪种？（列表页 / 左树右表 / 编辑页）
2. 我有没有用 Element UI 的原生组件？（el-card / el-table / el-form / el-button / el-dialog）
3. 我有没有用 prototype-standard.css 的 CSS 变量？还是写了硬编码色值？
4. 我有没有违反 H1-H10 反模式？
5. 我的删除按钮 / 危险操作 有没有二次确认？

5 个问题全部 ✅ 才能 commit。
